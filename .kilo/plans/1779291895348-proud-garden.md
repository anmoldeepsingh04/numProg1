# Project Plan: Numerical Programming 1 Tool

## Overview
This project aims to create an educational tool for the Numerical Programming 1 course at TUM. The tool will implement various numerical methods, provide visualizations (including live graphing), and be structured in an object-oriented manner for easy extensibility. The goal is to aid students in understanding the algorithms and performing well in exams.

## Current State
The repository already contains:
- A `numerics` directory with subdirectories for different numerical method categories (eigenValueApprox, interpolation, linearSolvers, nonLinearSolvers, numericalIntegration, utils).
- Each subdirectory contains some Python files implementing specific methods (e.g., bisection, secant, regula_falsi in nonLinearSolvers).
- A `utils/plotting.py` file for plotting.
- Basic setup files (requirements.txt, setup.py) and a tests directory.
- A minimal README.md.

## Objectives
1. Restructure the project to follow a clean, object-oriented design.
2. Implement a core library of numerical methods with a consistent interface.
3. Add live graphing capabilities to visualize the solution process.
4. Create a user-friendly interface (web-based or desktop) for students to interact with the tool.
5. Ensure the tool is easy to distribute and use (via GitHub, pip, or executable).
6. Write comprehensive documentation (README, docstrings, examples).

## Distribution Strategy
Given the user's novice status and the goal of maximum flexibility and best user experience, the recommended approach is:
1. **Primary Distribution**: A pip-installable package hosted on PyPI. This allows easy installation (`pip install numprog1-tool`) and updates.
2. **Secondary Distribution**: The GitHub repository with clear instructions for cloning and installing in development mode.
3. **Optional**: A Streamlit Cloud share link for instant web access without installation (if the UI is Streamlit-based).

This approach balances ease of use (students can install with one command) with flexibility (users can contribute, modify, and run in different environments).
